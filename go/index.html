Go packages

(of questionable quality)

Although python is definitely my language of choice right now for most things, I've been dabbling in go(lang), mostly out of distaste for c++, and a need for another high level compiled language I'm comfortable with.

My initial impressions of go are that:

  1. Static typing is nice for bigger apps (especially coming from python)

  2. The syntax is nice, coming from c++ and c

  3. Go's approach to packages is really neat. No need for venvs when go.mod keeps track of the specific versions of packages each project you're working on needs. And honestly, this just makes so much sense.

Anyways, as I expect I'll be putting together packages for various needs every now and then, this is where I'll be hosting them.

To use a package of mine, just type
import "burke.su/go/package-name"

My domain is easy to remember, and faster to type than a github / gitlab repo name, so I'm mostly hosting here out of convenience' sake. However, if it's a major project, especially something I am specifically available for more than just my own use, for the sake of enabling contributions I will link to a git repo (if not github than probably on my gitlab) where you can make pull requests or send a git request-pull.

That said, I also don't commit to making anything particularily noteworthy. Time will tell.

Below, you will find an index of my packages which are hosted here. Just for fun, I'll include a package with snippets of code from the process of my learning go, but for the rest, you can expect that if I'm posting it here I am committed to keeping it online and up-to-date.
